# RecruitOS v7 落地路线图

**基线**：PRD-v7.md + 四个维度改造方案  
**时间估算**：8-10 周（2 人团队）  
**交付节奏**：3 个 Batch，每个 Batch 有独立的里程碑和验收标准

---

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 真实数据质量差（面试评价文本短、标签稀疏） | 高 | 意向/校准模型效果低于预期 | 先用规则兜底，标注数据不足时降低 ML 模型权重 |
| LightGBM 训练数据不足（新租户） | 高 | 模型无法训练 | 行业模板冷启动，过渡期规则+ML 混合 |
| Feign 调用微服务超时 | 中 | AI 页面加载慢/空白 | 缓存 + 降级 + 超时 2s 返回规则结果 |
| LLM structured output 不稳定 | 中 | 需求诊断解析失败 | JSON Schema 校验 + 重试 3 次 + 规则 fallback |
| 前端改造影响现有功能 | 低 | 现有 Kanban/Pipeline 页面回归 | 增量改造，不改已有逻辑，只加组件 |
